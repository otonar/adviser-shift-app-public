## 2026-07-13

### 変更点
- **LINE 連携（設定画面）の不具合を修正**：外部ブラウザで「LINEと連携する」を押すと、LINE ログインは通るのに**連携済みにならない**（エラーも出ない）問題を解消。
  - ログインから戻ってきたときに**自動で連携を完了**するようにした（ボタンを押し直す必要なし）。
  - 連携に失敗した場合は**エラーメッセージを表示**するようにした。

### 対応
- `git pull` のみ。環境変数の変更・追加インストールなし。
- **LINE Developers Console 側の設定変更が必要**（管理者作業）：LINE ログインチャネルの LIFF の **Endpoint URL** を `https://adviser-shift-app.vercel.app/dashboard/settings` に変更し、Callback URL に同 URL（±末尾スラッシュ）が入っていることを確認。
- 本番（Vercel）は `main` への push で自動再デプロイ。

## 2026-07-01（その2）

### 変更点
- **パスワードのリセット／変更機能を追加**（パスワードを忘れたときの復旧手段）:
  - **管理者**: メンバー管理の各メンバーに「**PWリセット**」ボタンを追加。押すと**仮パスワードが1回だけ表示**されるので、本人に伝える（画面を閉じると再表示できません）。リセットするとロックも解除されます。
  - **本人**: 設定画面に「**パスワード変更**」を追加。仮パスワードで入った人が、現在のパスワードを入れて自分の好きなパスワードに変更できます（8文字以上）。
  - パスワードは引き続きハッシュ化して保存（管理者でも中身は見られません）。
- **セッション失効（token バージョニング）**: パスワードを**リセットするとその人の全端末**、**本人が変更すると他端末**が自動でログアウトされます（操作中の端末はそのまま）。Cookie を盗まれても、パスワード変更で締め出せます。

### 対応
- `git pull` のみ。環境変数の変更・追加インストールなし。
- **DB マイグレーション `006_add_token_version.sql` は本番 DB に適用済み**（共有 DB のため各自の作業は不要）。
- 本番（Vercel）は `main` への push で自動再デプロイ。
- ℹ️ この更新自体では既存のログインは維持されます（一斉ログアウトは起きません）。

## 2026-07-01

### 変更点
- **ログイン失敗カウンタのアトミック化**（セキュリティ堅牢化。通常利用の挙動は不変）:
  - ログイン失敗回数の加算を DB 関数側で**行ロックして直列化**するように変更。これまでは失敗リクエストを**同時に大量送信**するとカウントが正しく増えず、5回ロックを回避できる余地があった（総当たり対策の穴）。ユーザー・管理ログインの両方に対応。
  - 失敗時の残り回数表示・15分ロック・15分ウィンドウなど、**利用者から見える挙動は従来どおり**。

### 対応
- `git pull` のみ。**環境変数の変更・追加インストールなし**。
- **DB マイグレーション `005_atomic_login_failure.sql` は本番 DB に適用済み**（共有 DB のため各自の作業は不要）。
- 本番（Vercel）は `main` への push で自動再デプロイ。

## 2026-06-29（その3）

### 変更点
- **コードレビュー＆セキュリティ点検にもとづく堅牢化**（致命的な脆弱性は無し。挙動は不変）:
  - シフト枠作成・自動割り振り・手動調整・やり直しで、付随データの書き込み失敗時に**不完全な状態が残らない**よう、エラー検査と取り消し（補償削除）を追加。
  - 全 API の動的 URL（`/[id]`）に **UUID 形式チェック**を追加（不正なIDは一律「見つかりません」）。
  - 本番の **Content-Security-Policy から `unsafe-eval` を除外**（開発時のみ許可。XSS 時の攻撃面を縮小）。
  - JWT の署名・検証アルゴリズムを **HS256 に固定**（多層防御）。

### 対応
- `git pull` のみ。**DB スキーマ変更・環境変数の変更・追加インストールなし**。
- 本番（Vercel）は `main` への push で自動再デプロイ。
- ⚠️ CSP 変更があるため、デプロイ後にプレビュー/本番で**画面が正常に描画されること**（白画面でない・主要操作可）を一度確認してください。

## 2026-06-22（その2）

### 変更点
- **目安箱機能を追加**: スタッフが運営へ**質問・意見・相談**を投稿できる目安箱を新設。
  - 投稿フォーム：**種類**（商品/業務/人間関係/このアプリ/その他）・**区分**（質問/意見/相談/その他）・**名前**（表示／非表示）・**公開範囲**（全体／コアだけ）・**内容**。
  - **公開範囲「コアだけ」** は、研修役割に「コアメンバー」を持つスタッフだけが閲覧可能（管理者は全件閲覧可）。

  - スタッフ画面：自分が見られる投稿の一覧、管理画面：全投稿の閲覧＋**未対応／対応済みのステータス管理**＋削除。
  - 下ナビ・管理サイドバー・ダッシュボードに「目安箱」を追加。

### 対応
- ⚠️ **DB スキーマ変更あり**: `supabase/migrations/003_create_suggestions.sql` を**本番 Supabase に適用してください**（`suggestions` テーブルを新規作成）。**未適用だと目安箱がエラーになります**。
- 追加インストール・環境変数の変更は不要。
- 本番（Vercel）は `main` へのマージで自動再デプロイ。**migration 適用後にデプロイ**するのが安全。
- 備考: 「名前 非表示」でも投稿者の記録（user_id）はサーバーに保存され、不適切投稿時に運営が辿れる前提です（厳密な完全匿名ではありません）。
