# 変更履歴（CHANGELOG）

チームメンバー向けの変更まとめ。新しいものを上に追記する。
各エントリは「変更点」と、必要なら「対応（pull 以外に必要なこと）」を書く。
原則として `main` に反映した内容を記録する。

## 2026-06-22（その2）

### 変更点
- **目安箱機能を追加**: スタッフが運営へ**質問・意見・相談**を投稿できる目安箱を新設。
  - 投稿フォーム：**種類**（商品/業務/人間関係/このアプリ/その他）・**区分**（質問/意見/相談/その他）・**名前**（表示／非表示）・**公開範囲**（全体／コアだけ）・**内容**。
  - **公開範囲「コアだけ」** は、研修役割に「コアメンバー」を持つスタッフだけが閲覧可能（管理者は全件閲覧可）。
  - **名前「非表示」** は画面に投稿者名を出しません（管理画面でも非表示）。
  - スタッフ画面：自分が見られる投稿の一覧、管理画面：全投稿の閲覧＋**未対応／対応済みのステータス管理**＋削除。
  - 下ナビ・管理サイドバー・ダッシュボードに「目安箱」を追加。

### 対応
- ⚠️ **DB スキーマ変更あり**: `supabase/migrations/003_create_suggestions.sql` を**本番 Supabase に適用してください**（`suggestions` テーブルを新規作成）。**未適用だと目安箱がエラーになります**。
- 追加インストール・環境変数の変更は不要。
- 本番（Vercel）は `main` へのマージで自動再デプロイ。**migration 適用後にデプロイ**するのが安全。
- 備考: 「名前 非表示」でも投稿者の記録（user_id）はサーバーに保存され、不適切投稿時に運営が辿れる前提です（厳密な完全匿名ではありません）。

## 2026-06-22

### 変更点
- **在庫の更新日を表示**: 「在庫数がいつ更新されたか分からず、古い在庫を最新と思って案内してしまう」リスクへの対処。
  - スタッフの商品画面・管理の商品テーブルに「在庫更新: 6/22 14:30（3時間前）」のように**在庫数を最後に変えた日時**を表示。
  - 更新から **24 時間以上**経過した在庫は琥珀色＋「⚠️ 古い情報の可能性」で注意喚起。
  - 在庫の鮮度は名前・カテゴリ等の編集では更新されず、**在庫数を変えたときだけ**更新される（専用カラム `stock_updated_at`）。

### 対応
- ⚠️ **DB スキーマ変更あり**: `supabase/migrations/002_add_stock_updated_at.sql` を**本番 Supabase に適用してください**（`products` に `stock_updated_at` 列を追加。既存行は `updated_at` で初期化）。**未適用だと商品 API がエラーになります**。
- 追加インストール・環境変数の変更は不要。
- 本番（Vercel）は `main` へのマージで自動再デプロイ。**migration 適用後にデプロイ**するのが安全（先に列を用意 → コード配信）。
